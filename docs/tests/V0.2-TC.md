## 执行命令

### ⑤ 单元测试（无 SAP）

```powershell
cd "MCP Server"
pnpm test
```

### ⑥ E2E（需 SAP + user.txt）

```powershell
cd "MCP Server"
pnpm test:e2e
```

### HTTP 烟雾测试（T14、T15、E13、E14）

```powershell
cd "MCP Server"
node mcp-server.js
```

另开终端：

```powershell
# T14 — 服务信息
curl http://127.0.0.1:3000/

# T15 — initialize（记录响应头 Mcp-Session-Id）
curl -i -X POST http://127.0.0.1:3000/mcp `
  -H "Content-Type: application/json" `
  -H "Accept: application/json, text/event-stream" `
  -d '{"jsonrpc":"2.0","id":1,"method":"initialize","params":{"protocolVersion":"2024-11-05","capabilities":{},"clientInfo":{"name":"test","version":"1.0"}}}'

# T15 — tools/list（替换 <session-id>）
curl -X POST http://127.0.0.1:3000/mcp `
  -H "Content-Type: application/json" `
  -H "Accept: application/json, text/event-stream" `
  -H "Mcp-Session-Id: <session-id>" `
  -d '{"jsonrpc":"2.0","id":2,"method":"tools/list","params":{}}'
```

启用 API Key（E13、E14、U-14-A6）时在 `.env` 设 `MCP_REQUIRE_API_KEY=true`，curl 追加：

```powershell
-H "Authorization: Bearer <MCP_API_KEY>"
```

### stdio 工具验收（T9～T13、E9～E12）

```powershell
npx @modelcontextprotocol/inspector node mcp-server.js --stdio
```

或在 E2E 套件扩展后由 `pnpm test:e2e` 覆盖。
